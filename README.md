# 台灣股票技術分析推薦系統

以**歷史回測驅動**的台灣股票分析工具，整合為 Claude Code Skill。
輸入一個或多個台灣股票代號，自動分析技術指標 + 三大法人籌碼 + 市場多空趨勢，輸出未來一週的倉位建議：**加碼 / 持平 / 減碼**。

> **核心特色**：不使用固定的 if/else 規則，而是針對每支股票的過去 6 個月歷史數據，計算每個技術條件的實際預測準確率，再決定信號強度。同一個條件（如 RSI 超買）在不同股票上可能是截然相反的信號。並自動判斷台股大盤多空趨勢，在強多頭市場調高持倉偏向、提高減倉門檻，避免逆勢減碼。

## 安裝

### 前置需求

- Python 3.9 以上
- [Claude Code](https://claude.ai/code) CLI（用於 Skill）

### 步驟

```bash
# 1. Clone 專案
git clone https://github.com/YOUR_USERNAME/stock-recommender.git
cd stock-recommender

# 2. 安裝 Python 套件
pip install -r requirements.txt

# 3. 用 Claude Code 開啟此目錄
claude .
```

安裝完成。`.claude/commands/stock-analysis.md` 會被 Claude Code 自動載入為 `/stock-analysis` Skill。

## 使用方式

### 方式一：Claude Code Skill（推薦）

在 Claude Code 中輸入：

```
/stock-analysis 2330
/stock-analysis 2330 2317 0050
```

Claude 會執行分析並以中文提供：
- 技術指標快照（KD / RSI / MACD / 均線 / 乖離率）
- 總體市場指標（VIX 恐慌指數 / 巴菲特指標 / **台股多空趨勢**）
- 三大法人近 5 日買賣超與連買連賣天數
- **歷史回測表格**（每個條件的實際預測準確率）
- 分析師評語與倉位建議

### 方式二：直接執行 Python Script

```bash
python3 tech_analysis.py 2330
python3 tech_analysis.py 2330 2317 0050
```

### 欄位解說

| 欄位 | 說明 |
|------|------|
| **次數** | 過去 6 個月這個條件出現幾次 |
| **觸發** | ★ = 今日觸發且計入評分；✓ = 觸發但同組有更強信號 |
| **上漲率** | 條件觸發後，5 個交易日內股價上漲 >2% 的歷史比率 |
| **下跌率** | 條件觸發後，5 個交易日內股價下跌 >2% 的歷史比率 |
| **超額邊際** | `(上漲率 - 下跌率) - 基準值`。正值 = 此條件對這支股票歷史上偏多；負值 = 偏空 |
| **結論** | 強多↑↑ / 多↑ / 中性 / 空↓ / 強空↓↓（依超額邊際與樣本數判斷） |

### 建議標籤說明

| 建議 | 含義 | 對應倉位操作 |
|------|------|------------|
| **強力加碼** 🚀 | 多個高機率信號同向，技術面強勢 | 增加持倉至 75–100% |
| **加碼** ↑ | 技術面偏多 | 增加持倉 25% |
| **持平** → | 信號混合或無顯著信號 | 維持現有倉位 |
| **減碼** ↓ | 技術面偏空 | 減少持倉 25% |
| **強力減碼** ⚠ | 多個高機率空頭信號 | 大幅降低持倉至 0–25% |

### 為什麼同一個條件對不同股票結論不同？

這是系統的核心設計。以「RSI 超買(>70)」為例：

| 股票 | 超額邊際 | 結論 |
|------|---------|------|
| 2330（台積電） | -0.075 | 中性（超買不代表會跌） |
| 2317（鴻海） | **-0.552** | **強空**（超買後幾乎必跌） |

固定規則「RSI>70 就減分」會誤判 2330；本系統根據各股歷史數據各自學習。

## 計分邏輯

### 技術分數（Data-Driven）

1. 計算過去 6 個月每個交易日的所有技術條件
2. 計算每個條件的「**超額邊際**」= 條件的 (上漲率 − 下跌率) − 基準值（什麼都不做的自然勝率）
3. 每個指標類別（KD / MACD / RSI / 5MA / 月線 / 季線 / 200MA / 成交量 / 複合）只取**今日最強的一個條件**計分，避免相關條件重複累加
4. **技術分數** = 各組得分之和（每組 = 超額邊際 × 信心係數，信心係數隨樣本數增加）

### 籌碼分數（三大法人，固定規則）

| 條件 | 分數 |
|------|------|
| 外資連買 ≥ 3 日 | +2 |
| 外資連買 1-2 日 | +1 |
| 外資連賣 ≥ 3 日 | -2 |
| 外資連賣 1-2 日 | -1 |
| 投信連買 ≥ 3 日 | +2 |
| 投信連賣 ≥ 3 日 | -2 |
| 外資 + 投信同向買超 | +1 |
| 外資 + 投信同向賣超 | -1 |

### 市場多空趨勢調整（新）

系統自動抓取加權指數（^TWII）與其 MA200，判斷台股大盤所處的多空環境，並在評分中加入偏移量：

| TWII / MA200 | 市場趨勢 | 分數調整 | 額外規則 |
|-------------|---------|---------|---------|
| ≥ 1.05（高 5% 以上） | 強多頭 | **+0.25** | 減碼門檻 -0.40、強力減碼 -0.80、移動閾值 2%、起始倉位 75% |
| ≥ 1.00 | 多頭 | +0.15 | — |
| ≥ 0.95 | 中性 | 0 | — |
| < 0.95 | 空頭 | -0.15 | — |

**強多頭的特殊處理**（只在 TWII 比 MA200 高 5% 以上時啟動）：
- 起始倉位 75%（一般市場為 50%）
- 減碼門檻收緊至 -0.40（一般 -0.25），需要更強的空頭信號才執行減倉
- 移動閾值調高至 2%（一般 1.5%），過濾雜訊，只對有意義的波動反應

### 最終建議門檻

```
合計 = 技術分數 + 籌碼分數 × 0.12 + 市場趨勢調整

≥ +0.60  →  強力加碼 🚀
≥ +0.25  →  加碼     ↑
           （持平區間）
≤ -0.25  →  減碼     ↓    （強多頭時門檻為 -0.40）
≤ -0.60  →  強力減碼 ⚠   （強多頭時門檻為 -0.80）
```

## 支援的技術條件

**KD 隨機指標**：黃金交叉、死亡交叉（含超賣 / 超買區判斷）、K 值低位回升、高位回落

**MACD**：黃金交叉（零軸上 / 下）、死亡交叉（零軸上 / 下）、柱狀圖擴大縮小（多頭 / 空頭動能）、零軸上下方

**RSI**：超賣(<30)、偏低(30-40)、中性(40-60)、偏高(60-70)、超買(>70)、極度超買(>80)、低位回升、高位回落

**移動平均線**（5MA / 20MA月線 / 60MA季線 / 200MA）
- 每條均線的「站上」「跌破」「突破(金叉)」「跌破(死叉)」共 4 個條件

**成交量**：爆量上漲、爆量下跌、縮量上漲、縮量下跌

**複合條件**：KD 超賣 + MACD 金叉、KD 超買 + MACD 死叉、跌破月線 + MACD 死叉 等

## 3 年倉位策略回測

`backtest_annual.py` 模擬過去 3 年，每天依 skill 信號調整倉位，驗證策略實際績效。

```bash
python3 backtest_annual.py 2330
python3 backtest_annual.py 2317
```

### 倉位模型

策略不做 all-in / all-out，改用 5 個倉位等級：

| 信號 | 倉位等級變化 | 範例（從 50% 出發） |
|------|------------|-------------------|
| 強力加碼 | +2 級 | 50% → 100% |
| 加碼 | +1 級 | 50% → 75% |
| 持平 | 不動 | 50% → 50% |
| 減碼 | -1 級 | 50% → 25% |
| 強力減碼 | -2 級 | 50% → 0% |

起始倉位：50%（中性）；若大盤判斷為**強多頭**，自動調整為 75%。倉位上限 100%、下限 0%（不做空）。

### 回測設計

- **Walk-Forward（無未來偷看）**：每個交易日僅使用前 125 個交易日（約 6 個月）的數據計算信號
- **交易成本**：手續費 0.1425%（買+賣）+ 證交稅 0.3%（賣方）
- **執行時機**：當日收盤後產生信號，隔日收盤執行
- **不含三大法人數據**：歷史籌碼 API 難以批量回溯，僅使用技術面信號

### 回測結果（2023-06 至 2026-06，含市場多空趨勢優化）

| 股票 | 策略總報酬 | 買進持有 | 年化報酬 | Sharpe | 最大回撤 | 換倉次數 | 平均持倉 |
|------|----------|---------|---------|--------|---------|---------|---------|
| **2330 台積電** | **+326%** | +327% | +66% | **2.03** | -17% | 52次 | 81% |
| **2344 華邦電** | **+312%** | +691% | +64% | 1.22 | -53% | 68次 | 87% |
| **2408 南亞科** | **+207%** | +534% | +48% | 1.02 | -63% | 50次 | 70% |

> **2330 亮點**：策略總報酬 +326%，與買進持有 +327% 幾乎持平，但最大回撤只有 -17%（買進持有 -31%），Sharpe Ratio 高達 2.03，且換倉次數僅 52 次（vs 未優化前的 160 次）。

**優化演進對比（2330 台積電）：**

| 版本 | 總報酬 | Sharpe | 換倉次數 | 平均持倉 |
|------|--------|--------|---------|---------|
| 原始版 | +85% | 1.23 | 160次 | 44% |
| + 市場多空趨勢 | +182% | 1.61 | 138次 | 71% |
| + 強多頭三項優化 | **+326%** | **2.03** | **52次** | **81%** |

## 注意事項

- **僅供參考**：本工具為技術分析輔助，不構成投資建議
- **歷史不代表未來**：回測基於過去歷史，市場結構改變後結論可能失效
- **樣本數限制**：出現次數少（< 4 次）的條件會被排除；次數少但邊際高的條件，信心係數較低
- **三大法人速度**：從證交所平行抓取，每次執行約 5–15 秒
- **支援股票**：台灣上市（.TW）及上櫃（.TWO）股票，含 ETF（0050、0056、00878 等）
- **不支援做空**：策略最低倉位為 0%，不涉及放空操作

## 常見問題

**Q：找不到某股票的數據？**
系統自動嘗試 `.TW`（上市）再嘗試 `.TWO`（上櫃）。若仍找不到，請確認股票代號正確。

**Q：三大法人數據抓取很慢？**
需回溯抓取約 20 個交易日，已使用 `ThreadPoolExecutor` 平行抓取，通常 5–15 秒完成。

**Q：可以分析 ETF 嗎？**
可以。0050、0056、00878 等均支援，且各有獨立的歷史回測統計。

**Q：回測結果可以直接用來操作嗎？**
不建議。回測有倖存者偏差（僅取得有數據的股票）、未考慮流動性和滑價，僅供評估策略方向參考。

**Q：需要 API Key 嗎？**
不需要。所有數據均來自公開免費 API。
